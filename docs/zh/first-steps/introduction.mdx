Stagehand 是一个浏览器自动化框架，可通过自然语言与代码控制网页浏览器。通过将 AI 的能力与代码的精确性相结合，Stagehand 让 Web 自动化更灵活、可维护，而且真正可靠。

## 浏览器自动化的问题

像 Playwright 和 Puppeteer 这样的传统框架会迫使你编写脆弱的脚本，一旦 UI 变化就容易崩。Web 代理承诺用 AI 解决这个问题，却让你受制于不可预测的行为。

**你被夹在两个糟糕的选择之间：**

- **过于脆弱**：传统选择器在网站变更时就会失效
- **过于“代理”**：AI 代理行为不可控且几乎无法调试

<CodeGroup>
  ```typescript TypeScript
  // Act - Execute natural language actions
  await page.act("click the login button");

  // Extract - Pull structured data
  const { price } = await page.extract({
    schema: z.object({ price: z.number() })
  });

  // Observe - Discover available actions
  const actions = await page.observe("find submit buttons");

  // Agent - Automate entire workflows
  const agent = stagehand.agent({
      provider: "anthropic",
      model: "claude-sonnet-4-20250514",
      options: {
        apiKey: process.env.ANTHROPIC_API_KEY,
      },
  })
  await agent.execute("apply for this job");
  ```

  ```python Python
  # Act - Execute natural language actions
  await page.act("click the login button")

  # Extract - Pull structured data
  result = await page.extract(
    schema={"price": float}
  )

  # Observe - Discover available actions
  actions = await page.observe("find submit buttons")

  # Agent - Automate entire workflows
  await agent.execute("apply for this job")
  ```
</CodeGroup>

## 为什么开发者选择 Stagehand

- **精确控制**：将 AI 驱动的操作与确定性代码相结合。你来决定需要多少 AI 参与。

- **真正可复现**：精确保存并回放动作。浏览器自动化不再有“只在我电脑上能跑”的问题。

- **可规模化维护**：一个脚本可自动化多个网站。网站变更时，你的自动化会自适应调整。

- **可组合工具**：通过 act()、extract()、observe() 和 agent() 自由把控自动化层级。
