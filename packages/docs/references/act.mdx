### Method Signatures

<Tabs>
<Tab title="TypeScript">

```typescript
// String instruction
await page.act(instruction: string): Promise<ActResult>

// With ActOptions
await page.act(instruction: string, options: ActOptions): Promise<ActResult>
```

**ActOptions Interface:**
```typescript
interface ActOptions {
  model?: ModelConfiguration;
  variables?: Record<string, string>;
  timeout?: number;
  page?: PlaywrightPage | PuppeteerPage | PatchrightPage | Page;
}
```

**ModelConfiguration Type:**
```typescript
type ModelConfiguration =
  | string  // Format: "<provider>/<model>" (e.g., "openai/gpt-5", "anthropic/claude-sonnet-4-5")
  | {
      modelName: string;
      apiKey?: string;
      baseURL?: string;
    };
```

</Tab>
<Tab title="Python">

```python
# String instruction
await page.act(instruction: str) -> ActResult

# With parameters
await page.act(
    instruction: str,
    model: ModelConfiguration = None,
    variables: Dict[str, str] = None,
    timeout: int = None,
    page: Any = None,
) -> ActResult
```

</Tab>

### Parameters

<ParamField path="instruction" type="string" required>
  Natural language description of the action to perform.
</ParamField>

<ParamField path="model" type="string | object" optional>
  Model configuration for this action. Can be:
  - A string in the format `"<provider>/<model>"` (e.g., `"openai/gpt-4o"`, `"anthropic/claude-3-7-sonnet-latest"`, `"google/gemini-2.0-flash"`)
  - An object with the following properties:
</ParamField>

<Expandable title="Model Configuration">
  <ParamField path="modelName" type="string" required>
    The model identifier in the format `"<provider>/<model>"`.
  </ParamField>

  <ParamField path="apiKey" type="string" optional>
    API key for the model provider. Overrides the default API key.
  </ParamField>

  <ParamField path="baseURL" type="string" optional>
    Custom base URL for the model provider API.
  </ParamField>
</Expandable>

<ParamField path="variables" type="Record<string, string>" optional>
  Key-value pairs for variable substitution using `%variable%` syntax. Prevents sensitive data from appearing in logs.
</ParamField>

<ParamField path="timeout" type="number" optional>
  Maximum time in milliseconds to wait for the action to complete.
</ParamField>

<ParamField path="page" type="PlaywrightPage | PuppeteerPage | PatchrightPage | Page" optional>
  Specific page instance to perform the action on. If not provided, uses the current page.
</ParamField>

### Returns `Promise<ActResult>`

<ParamField path="success" type="boolean" required>
  Whether the action was completed successfully.
</ParamField>

<ParamField path="message" type="string" required>
  Details about the action's execution.
</ParamField>

<ParamField path="actionDescription" type="string" required>
  Description of the action that was performed.
</ParamField>

<ParamField path="actions" type="Action[]" required>
  Array of executed actions with details.

  **Action Interface:**
  ```typescript
  interface Action {
    selector: string;
    description: string;
    method?: string;
    arguments?: string[];
  }
  ```
</ParamField>

```Example Response
{
  success: true,
  message: 'Action performed successfully',
  actionDescription: 'Click the sign in button',
  actions: [
    {
      selector: 'button[data-testid="sign-in"]',
      description: 'Sign in button at the top right of the page',
      method: 'click',
      arguments: []
    }
  ]
}
```

### Error Types

- **TimeoutError** - Action exceeded timeout limits
- **ElementNotFoundError** - Target element could not be located  
- **ActionFailedError** - Action could not be completed
- **StagehandError** - General Stagehand-specific errors
